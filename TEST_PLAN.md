## שלב 1 — תיקוני ליבה ✅ בוצע

### 1.1 מודל מחירים — Backend
**תרחיש:** צור קבוצת אירוע עם 5 משתתפים → הפעל → בדוק מחיר.

| פעולה | תוצאה צפויה |
|-------|------------|
| קבוצת אירוע, 5 משתתפים → הפעל | עלות: **15 ₪** |
| קבוצת אירוע, 10 משתתפים → הפעל | עלות: **20 ₪** |
| קבוצת אירוע, 15 משתתפים → הפעל | עלות: **30 ₪** |
| קבוצת אירוע, 20 משתתפים → הפעל | עלות: **35 ₪** |
| קבוצת אירוע, 40+ משתתפים → הפעל | עלות: **45 ₪** |
| קבוצה שוטפת, 5 משתתפים → הפעל | עלות: **49 ₪** |
| קבוצה שוטפת, 8 משתתפים → הפעל | עלות: **69 ₪** |
| קבוצה שוטפת, 10 משתתפים → הפעל | עלות: **79 ₪** |
| קבוצה שוטפת, 12+ משתתפים → הפעל | עלות: **89 ₪** |
| הארכת אירוע | עלות: **15 ₪** |

**בדיקת API:**
```
POST /api/groups/{id}/activate
→ response.amount_paid == מחיר נכון לפי טבלה
```

## שלב 2 — שדרוג Tier באמצע הפעלה ✅ בוצע

### 2.1 שדרוג אירוע
**תרחיש:** קבוצת אירוע הופעלה עם 5 חברים. חבר 6 מצטרף.

| פעולה | תוצאה צפויה |
|-------|------------|
| חבר 6 מצטרף | `join` מחזיר `tier_upgrade_required: true`, `upgrade_price_diff: 5` (20-15) |
| Admin פותח את הקבוצה | באנר כתום: **"נדרש שדרוג תוכנית — 5 ₪"** |
| Admin לוחץ "שדרג עכשיו" | נפתח ActivationScreen עם מחיר **5 ₪** |
| Admin משלם | הוצאת "ADL ShareFlow Service" נוצרת, באנר נעלם |

### 2.2 שדרוג שוטף
**תרחיש:** קבוצה שוטפת הופעלה עם 5 חברים (49 ₪). חבר 6 מצטרף.

| פעולה | תוצאה צפויה |
|-------|------------|
| חבר 6 מצטרף | `tier_upgrade_required: true`, `upgrade_price_diff: 20` (69-49) |
| Admin פותח את הקבוצה | באנר כתום: **"נדרש שדרוג תוכנית — 20 ₪"** |
| Admin לוחץ "שדרג עכשיו" | מחיר **20 ₪** |
| Admin משלם | snapshot מתעדכן ל-6, באנר נעלם |

### 2.3 ללא שדרוג — תוך תחום
**תרחיש:** קבוצת אירוע הופעלה עם 3 חברים (15 ₪). חבר 4 מצטרף.

| פעולה | תוצאה צפויה |
|-------|------------|
| חבר 4 מצטרף (עדיין ≤5) | **אין** `tier_upgrade_required` — שני המחירים זהים |
| הקבוצה ממשיכה רגיל | ללא באנר שדרוג |

### 2.3b אפשרויות חלוקת תשלום שדרוג
**תרחיש:** Admin נכנס למסך שדרוג — בדוק שמוצגות 2 אפשרויות כמו בהפעלה.

| פעולה | תוצאה צפויה |
|-------|------------|
| מסך שדרוג נפתח | מוצג אייקון ⬆️ |
| סעיף "חלק על כל חברי הקבוצה" | מסומן כברירת מחדל |
| לחיצה על "אני משלם לבד" | מתגבר ✅ |
| שדרוג עם "חלק" | הוצאת "ADL ShareFlow Service" מחולקת בין כל חברים |
| שדרוג עם "לבד" | הוצאה על Admin בלבד |
| שורת "תוקף" | **לא מוצגת** (אין שינוי בתוקף בשדרוג) |

### 2.4 שדרוג מגבול קיצוני
**תרחיש:** קבוצת אירוע עם 15 חברים (30 ₪). חבר 16 מצטרף.

| פעולה | תוצאה צפויה |
|-------|------------|
| חבר 16 מצטרף | `upgrade_price_diff: 5` (35-30) |
| API endpoint | `POST /api/groups/{id}/upgrade-tier` → 200 OK |

### 2.5 endpoint שדרוג ישיר
```
POST /api/groups/{id}/upgrade-tier
Headers: Authorization: Bearer {token}
Body: { "split_among_group": true }
→ 200: { pricing_tier, max_participants_snapshot, amount_paid }
→ 400 אם אין צורך בשדרוג
```

## שלב 5 — Push Notifications (FCM)

### 5.1 הרשמה לאישור הודעות (iOS)
| בדיקה | תוצאה צפויה |
|-------|------------|
| פתיחת האפליקציה לראשונה | דיאלוג "האפליקציה מבקשת לשלוח התראות" מופיע |
| אישור ההודעות | FCM token נשמר בשרת (בדיקה בלוגים) |
| דחיית ההודעות | האפליקציה ממשיכה לעבוד ללא crash |

### 5.2 FCM Token Registration
| בדיקה | תוצאה צפויה |
|-------|------------|
| התחברות למשתמש | `POST /notifications/fcm-token` נקרא עם token + platform='ios' |
| logout | `DELETE /notifications/fcm-token` נקרא + token נמחק מה-DB |
| Token rotation | `onTokenRefresh` שולח token חדש לשרת אוטומטית |

### 5.3 קבלת הודעות
| תרחיש | תוצאה צפויה |
|-------|------------|
| הוצאה נוספה (foreground) | Toast/Banner מופיע מיד בתוך האפליקציה |
| הוצאה נוספה (background) | הודעת מערכת מופיעה בדשבורד iOS |
| לחיצה על הודעה | אפליקציה נפתחת + מנווטת לקבוצה הרלוונטית |
| הודעת "קבוצה הופעלה" | חברים מקבלים push (לא המפעיל) |
